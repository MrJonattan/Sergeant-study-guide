# Section 220 — Incident Facilities and Resources

## P.G. 220-19: CIMS Incident Facilities

**Date Effective:** 08/01/25 | **Last Revision:** R.O. 66 | **Pages:** 6

### Purpose

To define and describe CIMS Incident Facilities.

### Facility Types

| Facility | Description | Key Rules |
|----------|-------------|-----------|
| **Incident Command Post (ICP)** | Where Command Element performs command function; central coordination point | Only **ONE** per incident (except nuclear); generally within Outer Perimeter |
| **Operations Section Post** | Where Operations Section chief directs tactical operations | Generally **NOT co-located** with ICP; within Outer Perimeter |
| **Intelligence/Investigations Section Post** | Where I/I Section chief directs intelligence/investigative operations | Generally **NOT co-located** with ICP or Operations Post |
| **Incident Base** | Central location for logistic support and uncommitted resources | **Not activated in NYC** (absent extraordinary circumstances) |
| **Camp** | Temporary location for food, water, sanitation, lodging | **Not activated in NYC** (absent extraordinary circumstances) |
| **Other Facilities** | Mass Care, Triage, Temporary Morgue, Family Assistance, Mass Decontamination | As needed |

### ICP Location Requirements

| Requirement |
|-------------|
| Reasonable/appropriate distance from incident |
| Safety, health, security, force protection |
| Easy/expeditious access and egress |
| Sufficiently large, allows expansion |
| Permits continuous operations |
| Adequate workspace and utility/communications |
| Generally within **Site Access Control / Outer Perimeter** |

### ICP Key Rules

- Initially, ICP is wherever the Command Element is located
- Can be any facility: vehicle, trailer, tent, open area, room
- Identified by **name of the incident**
- Marked with flag, banner, beacon, sign, or distinctive marker
- Should **NOT be relocated** if possible (but may be if conditions change)

> **Exam Alert:** **Incident Base** and **Camp** are NOT activated in NYC absent extraordinary circumstances. These are NIMS ICS concepts for incidents outside NYC.

## P.G. 220-20: CIMS Incident Facilities — Staging Area

**Date Effective:** 08/01/13 | **Pages:** 4

### Definition

**Staging Area** — A temporary location where resources are assembled, organized, and await assignment.

### Key Rules

| Rule | Detail |
|------|--------|
| Number | More than one Staging Area may be established |
| Naming | Each identified by **geographic name or number** |
| Manager | Each has a designated **Staging Area Manager** |
| Resource status | Resources at Staging Area are in **"Available" status** |
| Readiness | Resources must be ready for deployment within **3 minutes** |
| Location | Close enough for timely deployment but far enough for safety |
| Security | Adequate safety, health, security, force protection measures |

### Staging Area Manager Duties

- Manage resources in Available status
- Organize resources for deployment
- Track resource status
- Ensure Check-In is performed
- Coordinate with Operations Section chief

> **Exam Alert:** Resources in "Available" status at a Staging Area must be ready for deployment within **3 minutes**.

## P.G. 220-24: CIMS "Check-In" Process

**Date Effective:** 08/01/13 | **Pages:** 2

### Purpose

To describe the "Check-In" process at an incident.

### What Check-In Accomplishes

| Function |
|----------|
| Ensure resource accountability |
| Track number, location, and status of resources |
| Assign and reassign resources |
| Locate personnel for emergency notifications |
| Prepare accurate time and overtime records |
| Plan for demobilization |
| Demobilize and release resources |

### Who Performs Check-In

| Priority | Performer |
|----------|-----------|
| 1st | Resources Unit |
| 2nd | Planning Section chief (if Resources Unit not activated) |
| 3rd | Command Element (if Planning Section not activated) |

### Check-In Locations (NYC)

| Location |
|----------|
| **Staging Area(s)** — Check-In with Resources Unit or Staging Area Manager |
| **Incident Command Post** — Check-In with Resources Unit |
| **Intelligence/Investigations Section Post** — When applicable |
| **Helibase(s)** — Absent extraordinary circumstances, not used in NYC |

### Check-In Forms

**DETAIL ROSTER / ASSIGNMENT SHEETS (PD406-141)** or other appropriate forms.

### Critical Rule

Resources must formally **Check-In before** reporting to Operations Section for assignment (absent extraordinary circumstances).

> **Exam Alert:** Resources must **Check-In before assignment** to the Operations Section. The 4 NYC Check-In locations are: Staging Area, ICP, I/I Section Post, and Helibase.

## P.G. 220-25: CIMS Resources

**Date Effective:** 07/19/16 | **Pages:** 5

### Definitions

| Term | Definition |
|------|-----------|
| **Resources** | Personnel, teams, equipment, supplies, vehicles, aircraft, watercraft, facilities |
| **Kind** | Broad class characterizing similar resources (e.g., marked patrol car, van, motorcycle) |
| **Type** | Level of capability: Roman numeral I (greatest) to IV (least) |

### Resource Categories

| Category | Description |
|----------|-------------|
| **Single Resource** | One vehicle/watercraft/aircraft + crew, single personnel, K-9 team, or pre-designated team (Squad, Mobile Field Force) |
| **Resource Team** | Same Kinds and Types of Single Resources assembled for a tactical assignment; must have designated Leader, transportation, common communications |
| **Special Response Team** | Any combination of different Kinds and Types assembled for a tactical assignment; must have Leader, transportation, common communications |

### NYPD Resource Examples

| Resource | Composition |
|----------|-------------|
| **Squad** | 1 Sergeant + 8 Police Officers |
| **Mobile Field Force** | 2 to 6 Squads |
| RMP | 1 vehicle + 2 Officers |
| Highway Motorcycle Squad | 1 Sergeant + 5 Officers on motorcycles |
| Mounted Squad | 1 Sergeant + 5 Mounted Officers on horses |
| ESU Entry Team | 1 Sergeant + 7–8 ESU Officers + vehicle(s) |
| ESU CAT Car | Nondescript Suburban + 1 Sergeant + 4 Officers |
| Bomb Squad Response Team | Van + 2 Officers + EDC + RMI |

### Resource Status Conditions

| Status | Description |
|--------|-------------|
| **Assigned** | Performing work tasks under supervisor direction |
| **Available** | Ready for assignment; deploy within **3 minutes** |
| **Out-of-Service** | Not able to be assigned or available (mechanical, rest, illness, injury, etc.) |

### Status Change Authority

Command Element, Operations Section chief(s), Branch Directors, Sector Supervisors, Group Supervisors, Resource Team/Special Response Team Leaders, I/I Section chief, Unit Leaders, Managers.

> **Exam Alert:** The 3 resource statuses are **Assigned, Available, Out-of-Service**. Available resources deploy within **3 minutes**.

> **Exam Alert:** A **Squad** = 1 Sergeant + 8 Officers. A **Mobile Field Force** = 2 to 6 Squads.
