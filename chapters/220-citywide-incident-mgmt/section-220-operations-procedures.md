# Section 220 — Incident Operations and Procedures

## P.G. 220-14: CIMS Supporting Agencies and Agency Representatives

**Date Effective:** 08/01/13 | **Pages:** 2

### Purpose

To define Supporting Agencies and Agency Representatives.

### Definition

**Supporting Agency** — An agency that is NOT a Primary Agency but has relevant resources, information, expertise, or jurisdictional authority.

**Agency Representative** — A representative designated by a Supporting Agency to function as the agency's spokesperson at the incident.

### Key Rules

- Agency Representative reports to the **Liaison Officer** (or Command Element if no Liaison Officer)
- Must have **authority** to make decisions on behalf of their agency
- All Supporting Agencies co-locate at the ICP when directed

## P.G. 220-15: CIMS Coordinating Agency

**Date Effective:** 08/01/25 | **Last Revision:** R.O. 66 | **Pages:** 1

### Definition

**NYC Emergency Management (NYCEM)** is New York City's designated **Coordinating Agency**.

### Key Roles

- Facilitate, coordinate, and support inter-agency activities
- May respond to multi-agency incidents
- Participate in Unified or Single Command organization
- Facilitate and support **Recovery/Restoration Operations**
- Ensure ICP and Operations Section Post are established and staffed

> **Exam Alert:** **NYCEM** is the Coordinating Agency — they facilitate and support, but do NOT command the incident.

## P.G. 220-16: CIMS Integrated Communications

**Date Effective:** 08/01/25 | **Last Revision:** R.O. 66 | **Pages:** 2

### Key Points

- Use **clear text** for inter-agency communications (no codes)
- Use **CIMS nomenclature/terminology**
- Observe strict radio and phone communications discipline
- Develop and implement a **Communications Plan** (component of the IAP)
- When appropriate, use Borough/Citywide interoperable radio frequencies
- NYC Agencies' Citywide interoperable **800 MHz** radio frequency

> **Exam Alert:** All incident personnel must use **clear text** (no 10-codes) for inter-agency communications and use **CIMS terminology**.

## P.G. 220-17: CIMS Transfer of Command

**Date Effective:** 02/04/15 | **Pages:** 2

### Purpose

To describe the formal Transfer of Command process.

### Options Upon Arrival of Ranking Member

| Option | Action |
|--------|--------|
| 1 | Assume command (if no Command Element exists) |
| 2 | Assume command from current Command Element |
| 3 | Make **no change** to current Command Element |
| 4 | Designate a **third party** to assume command |

### When Transfer of Command Occurs

- Higher-ranking person arrives and elects to assume command
- Incident stabilized — transfer to less senior person
- New Operational Period — designated personnel assume command
- Command Element dismissed at end of tour

### Transfer of Command Checklist

| Item |
|------|
| CIMS organization implemented? CIMS terminology in use? |
| One single ICP established? |
| Face-to-face transfer if possible? |
| Incoming Command Element personally assesses situation with outgoing? |
| Comprehensive briefing provided? |
| Appropriate time chosen for formal transfer? |
| Incoming Command Element formally accepted command? |
| Notifications made? |

### Transfer of Command Briefing Should Include

Concise history, current situation, objectives/strategy/priorities, IAP, organizational structure, resource assignments, status of tactical/investigative operations, intelligence/threat information, incident facilities status, communications plan, constraints/limitations, current concerns, future predictions, delegation of authority issues.

> **Exam Alert:** A ranking member arriving at an incident has **four options**: assume command, assume from current CE, make no change, or designate a third party. Transfer should be **face-to-face** when possible.

> **Sergeant Focus:** An arriving sergeant may elect to **make no change** to the command element if the current officer is managing effectively — assuming command is not mandatory.

## P.G. 220-18: CIMS Incident Sequence

**Date Effective:** 08/01/25 | **Last Revision:** R.O. 66 | **Pages:** 3

### Purpose

To describe the general CIMS Incident Sequence.

### Incident Sequence (In Order)

| Phase | Activities |
|-------|-----------|
| **1. Initial Response** | Agencies respond; first arrivals assess scene |
| **2. Life Safety Operations** | Immediate #1 priority; rescue, EMS |
| **3. Site Management Operations** | NYPD establishes perimeters; excludes non-essential personnel |
| **4. Investigative Operations** | May run concurrently with Life Safety (with CE approval); life safety concerns take precedence |
| **5. Recovery/Restoration Operations** | Final mitigation; return to non-emergency; may be delayed for investigation |
| **6. Demobilization** | Operations Section Post and ICP demobilize; incident concludes |

### Key Sequencing Rules

- **Life Safety** is always the immediate #1 priority
- Investigative Operations may run **concurrently** with Life Safety (life safety concerns take precedence)
- Recovery/Restoration may be **delayed** to minimize disturbance to crime scenes
- Recovery/Restoration **may begin concurrently** with Life Safety and/or Investigation (with CE approval)

> **Exam Alert:** The incident sequence is: Response → Life Safety → Site Management → Investigation → Recovery/Restoration → Demobilization. Phases may **overlap** with CE approval.

## P.G. 220-28: CIMS Incident Management Team (IMT) Activation/Deployment

**Date Effective:** 08/01/13 | **Pages:** 3

### Purpose

To activate and deploy an Incident Management Team at an emergency or planned event.

### Activation Authority

| IMT Type | Activated By |
|----------|-------------|
| Citywide IMT | Police Commissioner, First Deputy Commissioner, or Chief of Department |
| Borough IMT | Chief of Patrol, or Patrol Borough Commander (or Duty Chief) |
| Transit Bureau IMT | Chief of Transit (or Duty Chief) |
| Housing Bureau IMT | Chief of Housing (or Duty Chief) |

### Situations Requiring IMT Activation

| Situation |
|-----------|
| "Stand Alone" protocols activated |
| Incident extends into **multiple Operational Periods (>24 hours)** — formal written IAP required |
| Unified Command with multi-agency resources expected to exceed 24 hours |
| Declaration of Disaster |
| National Special Security Event (e.g., UNGA, RNC) |
| Incident of National Significance outside NYC with NYC impact |
| Substantial city/state/federal resources required |
| Natural disaster affecting NYC |

### Key Definitions

| Term | Definition |
|------|-----------|
| **Incident Management Team (IMT)** | Command Element + appropriate Command Staff + General Staff assigned to an incident |
| **Operational Period** | Period for execution of a given set of operations per the IAP; usually not over **24 hours** |
| **Incident Action Plan (IAP)** | Contains Incident Objectives, strategy, tactical actions for next Operational Period; may be oral or written |

> **Exam Alert:** When incident extends beyond **24 hours**, a formal **written IAP** is required. Operational Periods are usually not over 24 hours.

> **Exam Alert:** **Citywide IMT** activated by Police Commissioner, First Deputy, or Chief of Department. **Borough IMT** activated by Chief of Patrol or Patrol Borough Commander.
